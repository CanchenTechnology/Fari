# 今日神谕翻牌预生成流程

## 目标

点击“翻开今日卡牌”后，界面先进入一段准备时间。后续这里可以播放翻牌动画。

准备时间内先生成并缓存今日神谕需要的完整数据：

- 卡牌 icon
- 卡牌名字
- 卡牌正逆位
- 塔罗牌描述
- 今日神谕短解析
- 完整解读
- 传给对话系统的 TodayCardPayload

准备完成后，再显示卡牌容器、打开 `OracleReadingUI`，后续完整解读页也直接读取同一份缓存。

## 核心类

新增文件：

`Assets/Scripts/Dialog/Data/TodayOraclePreparedReading.cs`

这个类是一张今日牌的完整预生成结果：

```csharp
public class TodayOraclePreparedReading
{
    public TarotCard card;
    public bool upright;

    public string cardId;
    public string cardDisplayName;
    public string cardDescription;
    public string cardMeaning;
    public Sprite cardIcon;

    public TodayCardPayload cardPayload;
    public TodayOraclePayload oraclePayload;
    public CompleteInterpretationPayload interpretationPayload;

    public string preparedAt;
}
```

UI 后续不要各自重新拼数据，优先读取：

```csharp
DailyOracleService.Instance.CachedPreparedReading
```

## 调用流程

入口在：

`Assets/Scripts/UI/TodayOracleUI.cs`

点击按钮后会走：

```csharp
OnflipCardButtonClick()
```

内部启动协程：

```csharp
PrepareAndRevealTodayCardRoutine()
```

流程是：

1. 隐藏翻牌按钮。
2. 抽取并锁定今日牌。
3. 调用 `DailyOracleService.PrepareTodayReading(card, upright)`。
4. 打开 `LoadingTextUI`，显示类似“正在解读世界。。。”的打字机动画。
5. 等待 `flipRevealDelaySeconds` 和 AI 预生成都完成。
6. 关闭 `LoadingTextUI`。
7. 用 `TodayOraclePreparedReading` 赋值卡牌容器。
8. 同步 `DialogSystem.SetTodayCardPayload(...)`。
9. 打开 `OracleReadingUI`。

## 动画怎么接

`TodayOracleUI.cs` 里有这个字段：

```csharp
[SerializeField] private float flipRevealDelaySeconds = 1.2f;
```

后续如果翻牌动画是 2 秒，把 Inspector 里的值改成 `2` 即可。

如果动画结束需要主动通知，也可以把协程里的延迟判断替换成动画完成标记：

```csharp
while (!animationDone || !preparedReady)
{
    yield return null;
}
```

原则是：动画没播完不展示，AI 数据没准备好也不展示。

## LoadingTextUI

加载窗口脚本：

`Assets/Scripts/UI/LoadingTextUI.cs`

今日神谕翻牌时会调用：

```csharp
_loadingTextUI = UIModule.Instance.PopUpWindow<LoadingTextUI>();
_loadingTextUI.SetReadingCardText(card);
```

默认文案格式：

```text
正在解读{牌名}。。。
```

例如抽到世界时会显示：

```text
正在解读世界。。。
```

`LoadingTextUI` 内部会循环播放打字机动画。预生成完成后，`TodayOracleUI` 会调用：

```csharp
_loadingTextUI.HideWindow();
```

## 服务层方法

新增方法在：

`Assets/Scripts/Dialog/Data/DailyOracleService.cs`

```csharp
public void PrepareTodayReading(
    TarotCard card,
    bool upright,
    Action<TodayOraclePreparedReading> onReady = null)
```

它会同时准备：

- `RequestDailyOracle`
- `RequestCompleteInterpretation`
- `TarotSpriteLoader.Load(card.cardId)`
- `TodayCardPayload`

生成完成后写入：

```csharp
CachedPreparedReading
```

判断缓存是否属于当前牌：

```csharp
IsCachedPreparedReadingFor(card, upright)
```

## 后续页面怎么用

`OracleReadingUI` 已经改成优先读：

```csharp
DailyOracleService.Instance.CachedPreparedReading
```

如果缓存是同一张牌，就直接使用预生成的名字和图片。

`CompleteInterpretationUI` 已经改成优先读：

```csharp
CachedPreparedReading.interpretationPayload
```

这样点击“查看完整解读”时不会重复请求 AI。

## 注意

- `TodayOraclePreparedReading` 是本次今日牌的数据快照。
- 同一天同一张牌重复打开，优先复用缓存。
- 如果 AI 请求失败，`DailyOracleService` 原有逻辑会走 fallback，仍然会生成一份可展示的数据。
- 不要让 UI 分散调用多个 AI 入口，否则会出现名字、描述、完整解读不同步的问题。
