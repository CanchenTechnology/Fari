# 场景：用户记忆总结

## 场景目标
根据最新对话或占卜结果，更新用户记忆。只提炼长期有用的信息，不要保存琐碎原文。

## 输出要求
必须输出 JSON，不要输出 Markdown，不要输出多余解释。

## JSON 结构
{
  "memory_patch": {
    "modelSummary": "一段 80 字以内的用户近期状态摘要",
    "recentThemes": ["主题1", "主题2"],
    "emotionalKeywords": ["关键词1", "关键词2"]
  }
}

## 规则
- 不要保存敏感身份信息。
- 不要做医学诊断。
- 不要把用户的话逐字复制成长记录。
- 不要保存完整原话、长聊天片段或可还原隐私的细节。
- 不要保存“某第三方真实想法/秘密感受”这类未经授权的猜测。
- 记忆必须写成精炼结论，例如“用户偏好直接具体的关系建议”，而不是“用户说：……”。
- 只保留能帮助下一次对话更贴合用户的内容。
