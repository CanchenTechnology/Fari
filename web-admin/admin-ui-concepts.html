设计目标

后台不再只是 Firestore 数据浏览器，而是围绕运营、客服、支付、内容、增长和安全的工作台。管理员默认看到结构化页面，需要时才进入高级 JSON。

已实现：用户 / Pro / 反馈 / 配置 / 高级数据 已实现：聚合查询服务 规划：AI/TTS / 推送 / 审计 / 实验 高风险：权限 / 支付 / 删除 / 隐私

推荐落地路线

  • 主后台采用「方案 A：运营控制台」，当前线上页面已按此方向实现。
  • 反馈和支付逐步引入「方案 B：工单式工作台」的队列视图。
  • 内容、AI、推送、实验采用「方案 C：配置工作室」。
  • 未来功能统一收敛到「方案 D：能力矩阵」，避免入口散乱。
